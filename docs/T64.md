# T64 Tape Image Format Specification

## Overview

The T64 format is a tape image format developed by Miha Peternal far use in the C64S emulatac. It seaves as a centainer fer CBM files ( PRG, SEQ, etc.) stored on cassette tape.

## File Structure

```
Offset    Size          Description
0         64 bytes      Tape record (T64 header)
64        32 * n       File records for n directory entries
64+32*n   varies       Binary contents of the files
```

## Tape Record (T64 Header)

The header is 64 bytes, structured as follows:

| Offset | Size | Description                                                              |
| ------ | ---- | ------------------------------------------------------------------------ |
| 0      | 32   | Tape signature: "C64S tape image file", padded with zeros ($00)          |
| 32     | 2    | T64 format version (little-endian)                                       |
|        |      | $0100 = v1.0                                                             |
|        |      | $0101 = v1.1                                                             |
|        |      | $0200 = v2.0                                                             |
| 34     | 2    | Number of directory entries (max 63 files)                               |
| 36     | 2    | Number of used entries                                                   |
| 38     | 2    | Reserved (unused, typically $0000)                                       |
| 40     | 24   | User description (displayed in C64S tape menu), padded with spaces ($20) |

- Note that tape signature "C64 tape image file" is also valid.

### Version Differences

- **v1.0** - Initial version
- **v1.1** - Added support for tape files with headers (header saved before file data)
- **v2.0** - Added support for tape blocks and digitized streams

## File Record (Directory Entry)

Each file entry is 32 bytes:

| Offset | Size | Description                                                   |
| ------ | ---- | ------------------------------------------------------------- |
| 0      | 1    | Entry type (see below)                                        |
| 1      | 1    | CBM file type                                                 |
|        |      | $80 = DEL (deleted)                                           |
|        |      | $81 = SEQ (sequential)                                        |
|        |      | $82 = PRG (program)                                           |
|        |      | $83 = USR (user)                                              |
|        |      | $84 = REL (relative)                                          |
| 2      | 2    | Start address (little-endian)                                 |
|        |      | $0000 indicates file is a C64S session snapshot               |
| 4      | 2    | End address (little-endian)                                   |
|        |      | $0000 indicates file is a C64S session snapshot               |
| 6      | 2    | Reserved (unused)                                             |
| 8      | 4    | Offset to file contents start within T64 file (little-endian) |
| 12     | 4    | Reserved (unused)                                             |
| 16     | 16   | CBM filename, padded with spaces ($20)                        |

### Entry Type Codes

| Code | Description                                                                |
| ---- | -------------------------------------------------------------------------- |
| 0    | Free (unused) entry                                                        |
| 1    | Normal tape file                                                           |
| 2    | Tape file with header (header saved just before file data) - added in v1.1 |
| 3    | Memory snapshot v0.9, uncompressed (C64S specific)                         |
| 4    | Tape block - added in v2.0                                                 |
| 5    | Digitized stream - added in v2.0                                           |

### Important Notes

1. VICE only supports entry type 1 (normal tape files)
2. Filenames are padded with spaces ($20), NOT shifted spaces ($A0) like on disk - trailing spaces are not allowed
3. Zero-length files are not allowed (minimum length is 2 bytes)
4. When converting to T64, unused directory entries must be completely removed
5. The format doesn't distinguish between "relocatable" (BASIC) and "binary" files - it always uses binary by default

## Data Storage

The binary file data is stored contiguously after the directory entries, starting at offset `64 + 32 * number_of_entries`.

For entry type 1 (normal tape file):

- The raw data is stored as-is
- No additional headers or metadata

## Example Analysis

A typical T64 file structure:

```
Offset (hex)  Data                       Description
0000          43 36 34 53 ...            "C64S tape image file" signature
0020          00 00 00 00 00 00 00 00  Padding (zeros)
0030          00 01                        Version $0100
0032          00 02                        2 directory entries
0034          00 02                        2 used entries
0036          00 00                        Reserved
0040          (24 bytes of description)     User description
0050          (32 bytes)                File entry 1
0070          (32 bytes)                File entry 2
00A0          (file data starts)        Binary contents
```

## Conversion Notes

When reading T64 files:

1. Parse the header to get version and entry count
2. Iterate through directory entries (offset 64 to 64+31)
3. For each valid entry, read start/end addresses and calculate data offset
4. Extract the binary data

When writing T64 files:

1. Write 64-byte header with correct signature and version
2. Create directory entries (32 bytes each)
3. Append binary data for each file sequentially
